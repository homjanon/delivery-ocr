# 送货单识别 · 本地工具 使用与打包说明

> 纯本地运行，零网页、零 CORS。复用网页版提示词 / 表头 / 合计逻辑。
> 支持 Win7（需 Python 3.9）与 Win10（Python 3.9 / 3.11 均可）。**用 Python 3.9 打的 exe（可在 Win10 上装 3.9 打包）Win7 与 Win10 通用**，详见下方「Win7 关键提醒」。

---

## 一、直接运行（需本机有 Python）

### 1. 安装 Python
- **Win10**：装 Python 3.11（或 3.9 都行）。官网 https://www.python.org/downloads/
- **Win7**：**必须装 Python 3.9.x**（最后一个支持 Win7 的版本，如 3.9.13）。
  安装时勾选 **Add Python to PATH**。
- 验证：命令行输入 `python --version`，能看到版本即成功。

### 2. 安装依赖
在 `ocr_delivery.py` 所在目录打开命令行，执行：
```bat
pip install -r requirements.txt
```
> requirements.txt 已锁版本，确保 Win7 + Python 3.9 也能装（pillow 固定 <10）。

### 3. 配置密钥
把 `.env.example` 复制为 `.env`，填入你的 Key，并选模型：
```ini
PROVIDER=zhipu            # zhipu / qwen35b / sensenova
ZHIPU_API_KEY=你的key
SILICONFLOW_API_KEY=
SENSENOVA_API_KEY=
```
`.env` 放在 `ocr_delivery.py` **同目录**即可（程序会自动读取）。

### 4. 运行（三种方式）
```bat
:: ① 识别整个文件夹（批量，最常用）
python ocr_delivery.py "D:\送货单\2026-07-15"

:: ② 指定若干张
python ocr_delivery.py a.jpg b.jpg

:: ③ 不带参数 → 弹文件选择框点选
python ocr_delivery.py
```
运行结束后在同目录生成 `送货单_YYYY-MM-DD.xlsx`。

### 5. 桌面 GUI 版（gui_delivery.py，推荐日常用）
如果你不想敲命令行，用这个**窗口程序**更顺手：下拉选模型、窗口里填 Key（可一键存到 `.env`）、点按钮选图、实时看表格预览、再点导出。
```bat
python gui_delivery.py
```
- 首次打开会自动从 `.env` 预填对应模型的 Key；也可直接在窗口里填，点「存Key到.env」持久化。
- 识别在后台线程执行，窗口不卡；状态栏实时显示耗时。
- 适合 Win7 / Win10，零额外依赖（Tkinter 随 Python 自带）。

## 二、打包成 exe（双击即用，免装 Python）

> 适合给同事或自己长期用。**注意 Win7 兼容性见下方红字。**

### 1. 装打包工具
```bat
pip install pyinstaller==5.13.2
```
> **为什么锁 5.13.2？** PyInstaller 6.x 在部分 Win7 环境启动有问题；5.13.2 对 Win7 最稳。

### 2. 打包
在 `ocr_delivery.py` 同目录执行：
```bat
pyinstaller --onefile ocr_delivery.py
```
- 打包完后，`dist\ocr_delivery.exe` 就是可执行文件。
- 想双击不弹黑窗口（纯文件选择框体验），可加 `--noconsole`：
  ```bat
  pyinstaller --onefile --noconsole ocr_delivery.py
  ```
  > 但加 `--noconsole` 后运行日志看不到，出错时不易排查，建议先不加。
- **GUI 版打包**（窗口程序，推荐双击用）：
  ```bat
  pyinstaller --onefile --noconsole gui_delivery.py
  ```
  GUI 版建议直接 `--noconsole`（无黑窗口，纯窗口体验）。产物 `dist\gui_delivery.exe`，与 `.env` 同目录即可双击使用。

### 3. 使用 exe
- **命令行版**：把 `dist\ocr_delivery.exe` 和 `.env` 放同一文件夹，双击 → 弹文件选择框选图 → 生成 `送货单_日期.xlsx`。
- **GUI 版**：把 `dist\gui_delivery.exe` 和 `.env` 放同一文件夹，双击 → 窗口内选模型 / 填 Key / 选图 / 导出。
- 输出文件默认生成在 exe 所在文件夹。

### ⚠️ Win7 关键提醒（重点）
- **兼容性只取决于"打包用的 Python 版本"，不取决于"在哪台机器打包"。**
- **推荐做法：在 Win10 上装一个 Python 3.9（可与已有的 3.11 共存，用 `py -3.9` 调用），用它打包**；打出的 exe **Win7 和 Win10 通用**，直接拷到 Win7 双击即用，**Win7 端无需安装 Python**。
- ❌ 切勿用 Python 3.11 打包：3.11 依赖 Win7 没有的系统 API，exe 在 Win7 会报错。
- 架构要一致：通常都是 64 位，Win10(64) 打的 exe 放到 Win7(64) 没问题；若 Win7 是 32 位，须用 32 位 Python 3.9 打包。
- 杀毒软件可能误报 PyInstaller 打包的 exe，加白名单/信任即可。
- 若 Win7 双击报"缺少 vcruntime140.dll"，装一次微软 **VS2015–2019 运行库**（免费，一次性）即可。

## 三、常见问题

| 现象 | 处理 |
|------|------|
| 提示找不到 API Key | 检查 `.env` 是否在程序同目录、对应 `PROVIDER` 的 Key 是否已填 |
| HTTP 429 / 服务繁忙 | 程序已自动重试 3 次；仍失败多为模型限流，稍后重试或换模型 |
| JSON 解析失败 / 被截断 | 图片过多过大导致触及 max_tokens；减少张数或等模型恢复正常 |
| 商汤调用异常 | 商汤模型名/返回格式以官方文档为准，可用 `.env` 的 `SENSENOVA_MODEL` 覆盖；必要时在代码 `MODEL_PRESETS["sensenova"]` 核对 |
| 中文乱码 | 全程 UTF-8，正常不会出现；若终端乱码，用 `chcp 65001` 切 UTF-8 代码页 |

## 四、与网页版的区别
- 调用方由「浏览器」变为「本地 Python」，因此**无 CORS 限制**，商汤等也能接。
- API Key 存本地 `.env`，比浏览器 localStorage 更安全。
- 支持**整个文件夹批量**识别，一次出一张总表。
- 逻辑（提示词 / 表头 / 合计 / 解析容错）与网页版完全一致。
